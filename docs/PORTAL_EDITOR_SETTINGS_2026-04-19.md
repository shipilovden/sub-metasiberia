# Portal Editor Settings (2026-04-19)

Изменения редактора порталов:

- У объекта `Portal` скрыт выбор `Font`, так как это поле для портала не используется.
- Для портала открыт блок `Materials` с фиксированными слотами:
  - `Material 0 (Inner Rim)`
  - `Material 1 (Arch Body)`
  - `Material 2 (Outer Edge)`
  - `Material 3 (Portal Effect)`
  - `Material 4 (Threshold)`
- `portal.bmesh` теперь принудительно remap'ится так, чтобы редактор показывал только 5 реально полезных portal-материалов.
- Актуальное соответствие slot -> геометрия:
  - `Material 0 (Inner Rim)` -> внутренний обод.
  - `Material 1 (Arch Body)` -> основное тело арки.
  - `Material 2 (Outer Edge)` -> внешний кант.
  - `Material 3 (Portal Effect)` -> процедурная внутренняя плоскость портала.
  - `Material 4 (Threshold)` -> порог внизу.
- Для старых порталов с историческим 4-слотовым набором клиент автоматически достраивает:
  - `Material 4 (Threshold)` как независимую копию старого `Material 0 (Inner Rim)`.
- Скрытый batch, который раньше шёл как отдельная `Inner Wall`, теперь слит в `Material 2 (Outer Edge)`, чтобы в UI не было бесполезного slot'а.
- Для новых и старых порталов клиент теперь гарантирует наличие полного набора portal-материалов по умолчанию.
- `Material 3 (Portal Effect)` теперь реально влияет на цвет внутреннего shader-эффекта через tint.
- Поле `Target URL` для портала показывается как `Portal Target URL`, с placeholder для `sub://...`.
- Ссылка `Open URL in web browser` для порталов скрыта, потому что игровой переход использует `sub://` и выполняется проходом через портал, а не открытием во внешнем браузере.

Технические заметки:

- Дефолтная геометрия портала по-прежнему использует встроенный `portal.bmesh`.
- Внутренний эффект остаётся процедурным shader'ом `portal_frag_shader.glsl`; изменения касаются tint через material colour, а не замены самого эффекта.
